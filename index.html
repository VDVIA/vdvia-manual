30초 요약

이 문서 전체를 한 문단으로

회사의 뇌는 문서 폴더 하나(vdvia 저장소)다. 사람은 옵시디언이라는 창으로, AI(클로드)는 클로드 코드라는 창으로 같은 문서를 읽고 쓴다. 밤마다 자동으로 그날의 대화에서 지식이 추출돼 문서로 쌓이고, 문서들의 관계가 그래프로 갱신된다. 그래서 내일의 AI는 오늘보다 회사를 더 잘 안다. 당신이 할 일은 습관 3개뿐이다 — ① 클로드가 작업 전 문서를 읽었는지 확인하고 ② 세션이 끝나면 "암묵지 정리"라고 말하고 ③ 문서는 옵시디언에서 보고 고친다.

사람의 창
옵시디언 — 읽기·편집·그래프 탐색. 저장하면 15분 안에 모두에게 자동 반영
▲ 같은 문서를 위아래가 공유 ▼
회사의 뇌
vdvia 저장소 — 모든 문서·결정·자동화가 사는 단일 정본 (GitHub이 백업·역사 담당)
▲ 같은 문서를 위아래가 공유 ▼
AI의 창
클로드 코드 — 지시받은 일을 문서를 읽고 실행. 밤엔 자동화 봇으로 일함

우리가 푸는 문제

3인이 AI와 함께 대기업처럼 일하려면

VDVIA는 3명(Marco·John·Sol)이 브랜드를 글로벌로 운영합니다. 그게 가능하려면 AI가 단순 도구가 아니라 회사 맥락을 아는 동료여야 합니다. 그런데 AI에게는 태생적 문제가 셋 있습니다:

🧠
지식이 머리에만 있다
말로 한 지시·결정은 그 대화가 끝나면 AI에게 존재하지 않는 정보가 된다. 문서로 남은 것만 남는다.
🙈
AI가 안 읽고 일한다
관련 문서가 있어도 AI는 시키지 않으면 안 읽는다 → 잘못 진행 → 재작업 = 최대 토큰·시간 낭비.
🕰
문서가 낡는다
결정은 매일 바뀌는데 문서가 안 따라오면, AI는 낡은 정보로 자신 있게 일한다.
우리의 답: ① 모든 지식을 문서 하나의 저장소에 모으고 ② AI가 작업 전 반드시 관련 문서를 읽게 강제하고(읽기 게이트) ③ 매일 밤 자동으로 그날의 지식을 추출·연결해 문서를 최신으로 유지한다. 이 세 개가 아래 전체 시스템의 뼈대다.
2

네 가지 원리

모든 규칙은 이 넷에서 나온다
📌
① 뇌는 문서 하나에 — 단일 정본
같은 정보가 두 곳에 있으면 반드시 어긋난다. 모든 지식은 vdvia 저장소의 정해진 한 곳에만 산다(§13 정본 지도). 다른 곳엔 링크만.
👁
② 사람과 AI가 같은 걸 본다
사람의 화면(옵시디언)과 AI의 입력(클로드 코드)이 같은 파일이다. 그래서 사람의 이해와 AI의 이해가 어긋나지 않는다 — 뇌 동기화.
🏃
③ 완벽보다 95% 속도
100% 정확을 기다리지 않는다. 빨리 실행하고 기록하고 고친다. 틀린 건 밤 파이프라인과 감사가 잡는다.
📈
④ 지식은 매일 복리로
오늘의 대화·결정이 밤에 자동으로 문서가 되고 그래프로 연결된다. 일할수록 AI가 똑똑해지고, AI가 똑똑할수록 일이 싸진다.

모든 폴더 입구에는 CLAUDE.md라는 안내판이 있습니다 — 이 폴더가 뭐 하는 곳인지, 일하기 전 꼭 읽을 문서(필독 5개 이하), 하지 말 것, 산출물을 어디 저장하는지. 사람도 AI도 이 안내판부터 봅니다. 그래서 새 세션의 클로드도, 새로 온 사람도 길을 잃지 않습니다.

당신이 지킬 습관 3가지

시스템의 나머지는 전부 자동 — 사람 몫은 이것뿐
1
작업 전, 클로드가 읽었는지 확인한다 (읽기 게이트)
클로드는 일 시작 전 그 폴더의 필독 문서를 읽고 "📚 로드: ..." 한 줄을 보고하게 돼 있다. 첫 응답에 이 줄이 있는지 눈으로 확인. 없거나 엉뚱한 파일이면 바로 지적한다 — AI 내부를 몰라도 이 한 줄로 검수가 된다.
확인할 것: "📚 로드: ..." 줄 · 없으면 → "필독 읽었어?"
2
세션이 끝나면 "암묵지 정리"라고 말한다
이 한 마디로 오늘 대화의 결정·폐기·노하우·아이디어가 자동 분류되어 회사 문서로 저장된다. 잊어도 밤 러너가 한 번 더 훑지만, 직접 트리거하면 더 정확하다. 다음날 아침 슬랙 무누락 리포트에서 내 이름 옆 ✅를 확인.
세션 마감 시: "암묵지 정리"
3
문서는 옵시디언에서 보고 고친다
터미널·깃허브 웹을 몰라도 된다. 옵시디언에서 열고 고치고 저장하면 15분 안에 자동으로 모두에게 반영된다(충돌도 자동 회피). 폴더를 옮기거나 지우는 건 클로드에게 시킨다 — 링크·자동화가 같이 움직여야 해서.
읽기·고치기 = 옵시디언 · 시키기 = 클로드 · 옮기기·지우기 = 클로드에게 요청

옵시디언

설치 15분, 이후엔 그냥 쓰면 됨

설치·설정은 가이드 하나로 끝: 저장소의 00_compass/guides/obsidian-setup-guide.md (STEP 1~5, 15분). 핵심 설정 — 15분 자동 저장·동기화, 충돌 자동 회피(pull 우선), 링크는 마크다운 형식 — 은 가이드대로 하면 자동으로 맞춰집니다.

✅
이렇게 쓴다
문서 열기·읽기·고치기(Cmd+E 편집 전환) · 그래프 뷰로 연결 훑기 · 캔버스에 생각을 그리고 경로 복사해 클로드에게 "다듬어줘"(사람↔AI 왕복) · 이미지는 미리보기로 그냥 봄

하지 않는다

구조 유지 규칙 — 지도 = 실물

"안내판이 없는 폴더를 가리키던" 2026-07-01 사건의 재발 방지

원칙 하나: 지도(문서)와 실물(폴더·자동화)은 항상 같아야 한다. 아래는 그걸 지키는 규칙과, 어긋남을 자동으로 잡는 장치다.

규칙왜 (실제 사건)
① 폴더를 옮기면 그 자리에서 안내판(CLAUDE.md)도 함께 고친다. "이관 완료" = 안내판 갱신 + drift 0 + 자동화 경로 재지정, 셋 다.재편 완료 선언 후 안내판 16개가 23일간 옛 경로를 가리킴
② 자동 재생성물은 기본 gitignore. 자동화 커밋은 add -A 금지 — 산출물 경로만 명시.add -A가 다른 팀 수집기 3종을 오삭제해 하루 전량 실패
③ 만들기 전에 상위 규칙을 먼저 읽는다. 필독·라우팅 건너뛰지 않기.하위 폴더 세밀 규칙을 상위에서 덮어쓸 뻔
④ 문서 충돌은 한쪽을 지우지 말고 둘 다 적어 결정자에게.지쉘 소유권 결정문 vs README 충돌
⑤ 새 자동화는 §17 인벤토리 등재 + 생존신호 없이 "가동" 선언 금지.틱톡 어필리에이트가 3주간 아무도 모르게 휴면
🔍 drift_check — 매일 아침 자동 스캔
07:00 배치가 _health/drift_check.py로 8가지 표류 신호(죽은 링크·유령 경로·구 SSOT·지도vs실물·죽은 launchd·60_agency 분리 위반 등)를 훑는다. 문제 0이면 침묵, 있으면 #ai-알림 한 줄.
📥 _inbox — 모르면 추측 안 함
저장 위치가 애매하면 흩뿌리지 말고 00_compass/_inbox/에 격리. 아침 배치가 대기 건수를 보고 → 사람이 재분류. 목표 상태 = 항상 비어있음.
🏷 한 이름 = 한 실체
한 이름이 2개 이상을 가리키면 즉시 {이름}-relations.md 정본 신설, 각 폴더는 링크만. (예: "지쉘" 3실체 → zishel-relations.md)
🔒 60_agency 데이터 분리
대행(60_agency)과 레몬더마는 데이터·계정·발행처 상호 참조 금지. 계약서·PII·클라이언트 미공개 데이터는 Drive에, 레포엔 링크·상태만.

3인 각자의 맥에서 러너가 매일 그날의 세션 + Slack + Notion 변경분을 읽고, 아래 6분류로 저장소 정본에 나눠 넣는다. 분류 기준 원본은 00_compass/_tacit-extract.md.

분류언제(신호)정본 위치
① 결정"결정함" · 갈래길 선택 · 가설 채택00_compass/decisions/ 신규
② 폐기·금기시도했는데 안 통함 · 같은 실패 2회+decisions/ (폐기- 접두)
③ 판단 휴리스틱"이런 경우엔 이렇게" · 동일 판단 2회+playbook 3계층 (회사/John/Marco)
④ 시장·페르소나 시그널고객 반응·채널 성과·경쟁사 — 가설을 흔드는 데이터결론→strategy-state.md · 고객 원본 목소리→content-intel/voc/

핵심: 분류·저장으로 끝이 아니다. 그 팀 CLAUDE.md나 필독에 링크가 걸려야 다음 세션이 실제로 읽는다. 그래서 야간 러너는 새 결정·폐기를 해당 폴더 CLAUDE.md에 링크로도 반영한다(추가만, 삭제 금지). 지식그래프(§10)가 이 연결을 매일 갱신·검수한다.
